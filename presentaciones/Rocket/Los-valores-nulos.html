Definicion de Valores Nulos

"Un valor nulo significa que es un valor desconocido, indefinido o no inicializado."

"El valor nulo es un estado diferenciado que no es igual que ningún otro valor."

(IBM documentation)

¿Qué son los valores nulos?

Null (nulo) es un marcador especial usado en el lenguaje de consulta estructurado (SQL) para indicar que no existe un valor dentro de una base de datos; Este fue introducido por el creador del modelo relacional de bases de datos E. F. Codd,

Importancia de los valores nulos en las bases de datos.

Los valores nulos son un concepto esencial en las bases de datos relacionales, diseñados para mejorar la coherencia y mantener la integridad de los datos, y su importancia se remonta a los primeros días de los sistemas de bases de datos.

Restricciones NOT NULL

Todos los tipos de datos incorporados proporcionados con el gestor de bases de datos dan soporte a la presencia de valores nulos. Sin embargo, algunas reglas de negocio pueden dictar que siempre se debe proporcionar un valor.

(por ejemplo, cada empleado debe proporcionar información de contacto de emergencia)

Restricciones NOT NULL

La restricción NOT NULL se utiliza para asegurarse de que a una columna determinada de una tabla no se le asigne nunca el valor nulo. Una vez que se ha definido una restricción NOT NULL para una columna determinada, cualquier operación de inserción o actualización que intente colocar un valor nulo en esa columna fallará.

Impacto en la Integridad de los Datos

La integridad de los datos es la exactitud, integridad y confiabilidad de los datos.

Integridad Lógica e Integridad de la entidad

  1. Se trata de la racionalidad de los datos presentes en la base de datos relacional.
  2. Depende de la elaboración de claves primarias o valores exclusivos que clasifiquen los elementos de datos. El propósito es garantizar que los datos no se registren varias veces (es decir, que cada elemento de datos sea único) y que la tabla no tenga campos nulos.

¿En que casos se utilizan los valores nulos?


Se pueden producir valores nulos al hacer referencia a campos del mensaje que no existen, al acceder a columnas de base de datos para las que no que se ha facilitado ningún dato y al utilizar la palabra clave NULL, que suministra un valor literal nulo.

Presencia de valores nulos


-->Datos faltantes:

En situaciones donde no se dispone de un valor específico para un campo en particular, se utiliza el valor nulo para indicar que no se tiene información disponible para ese campo.

Presencia de valores nulos


-->Registros incompletos:

Cuando un registro se está creando o actualizando y no todos los campos requeridos se han completado, el valor nulo puede usarse para indicar que el campo no ha sido completado.

Presencia de valores nulos


-->Errores de entrada:

En casos donde se produce un error durante la entrada de datos, como un error de tipeo o una entrada incorrecta, el valor nulo puede utilizarse para indicar que el valor ingresado no es válido o que no se puede determinar el valor correcto.

Como Manejar Valores Nulos

Hay 3 maneras de manejarlos las cuales son:

1.eliminacion de registros

2. relleno de valores faltantes

3. Uso de valores predeterminados

1.Eliminacion de registros

El comando de software más adecuado para ello es MySQL DELETE. Con esta función se especifica el conjunto exacto de datos que hay que eliminar. Para ello, debes especificar los datos correspondientes y además indicar de qué tabla se van a eliminar
Ejemplo, si sequiere eliminar el cliente "Scott Pilgrim", se utiliza el siguiente comando:

DELETE FROM Clientes

WHERE Name = "Scott Pilgrim"

De estaforma, se elimina la linea correspondiente de la tabla "Cliente"

2. Relleno de valores faltantes

En esta parte hay 2 comandos, uno es para VISUALIZAR y otro es para RELLENAR los espacios NULL o vacios en una sola consulta
mysql> SELECT * FROM alumnos WHERE ciudad= ''

con este comando solo nos va a mostrar valores vacios de la tabla "alumnos", como es el caso del campo DOS

UPDATE alumnos SET ciudad= 'madrid' WHERE ciudad IS NULL OR ciudad= '';

Este comando actualiza la comuna "CIUDAD" de la tabla Alumnos, estableciendo su valor como "madrid" para aquellas filas sean NULL o vacias

3.Uso de valores predeterminados

El valor por defecto para un campo de una tabla se indica luego del tipo de dato mediante "DEFAULT valor", donde el valor por defecto para un campo de una tabla se indica luego del tipo de dato mediante.

Si un campo no tiene definido un valor por defecto, entonces Mysql lo determina de la siguiente manera:

  1. Si el campo soporta el valor NULL, se lo define con un DEFAULT NULL explícito.
  2. Si el campo se define como NOT NULL (NO soporta el valor NULL), depende de la versión de Mysql:
  3. Mysql <= 4.1: define el campo con un DEFAULT explícito usando el valor implícito según el tipo de dato del campo
  4. Mysql >= 5.0.2: define el campo SIN un DEFAULT explícito y es necesario evaluar la forma en que se agregan y modifican registros

Tipos de valores nulos

NULL en SQL: En SQL, NULL es un valor especial que se utiliza para representar la ausencia de un valor válido en una columna de una tabla.

Indica que no se ha proporcionado ningún valor para esa columna en una fila específica o que el valor es desconocido o no aplicable en ese momento.

Los valores NULL pueden aparecer en cualquier tipo de columna de una tabla,

Tipos de valores nulos

NaN en Python: NaN (Not a Number) es un valor especial que se utiliza en Python para representar resultados indefinidos o no numéricos en operaciones matemáticas.

Por ejemplo, divisiones por cero o cálculos que resultan en valores complejos pueden producir NaN.

Es importante tener en cuenta que NaN es un valor de punto flotante y no se puede comparar directamente con otros valores utilizando operadores de igualdad (==), ya que siempre dará como resultado False.

Tipos de valores nulos

Otros tipos de valores nulos en diferentes sistemas de bases de datos:

algunos sistemas de bases de datos pueden proporcionar otros tipos de valores nulos o valores especiales para representar la ausencia de datos.

En Oracle, se puede usar una cadena vacía ('') en lugar de NULL en algunas situaciones.

En Microsoft SQL Server, además de NULL, también se puede utilizar el valor '' para representar una cadena nula.
