Definicion de Valores Nulos

"Un valor nulo significa que es un valor desconocido, indefinido o no inicializado."

"El valor nulo es un estado diferenciado que no es igual que ningún otro valor."

(IBM documentation)

¿Qué son los valores nulos?

Null (nulo) es un marcador especial usado en el lenguaje de consulta estructurado (SQL) para indicar que no existe un valor dentro de una base de datos; Este fue introducido por el creador del modelo relacional de bases de datos E. F. Codd,

Importancia de los valores nulos en las bases de datos.

Los valores nulos son un concepto esencial en las bases de datos relacionales, diseñados para mejorar la coherencia y mantener la integridad de los datos, y su importancia se remonta a los primeros días de los sistemas de bases de datos.

¿En que casos se utilizan los valores nulos?


Se pueden producir valores nulos al hacer referencia a campos del mensaje que no existen, al acceder a columnas de base de datos para las que no que se ha facilitado ningún dato y al utilizar la palabra clave NULL, que suministra un valor literal nulo.

Presencia de valores nulos


-->Datos faltantes:

En situaciones donde no se dispone de un valor específico para un campo en particular, se utiliza el valor nulo para indicar que no se tiene información disponible para ese campo.

Presencia de valores nulos


-->Registros incompletos:

Cuando un registro se está creando o actualizando y no todos los campos requeridos se han completado, el valor nulo puede usarse para indicar que el campo no ha sido completado.

Presencia de valores nulos


-->Errores de entrada:

En casos donde se produce un error durante la entrada de datos, como un error de tipeo o una entrada incorrecta, el valor nulo puede utilizarse para indicar que el valor ingresado no es válido o que no se puede determinar el valor correcto.

Impacto de la Integridad de los Valores Nulos

Los valores nulos pueden afectar significativamente la integridad de los datos en una base de datos. Representan información desconocida o no aplicable que puede sesgar los resultados de consultas y análisis.

  • Los valores nulos introducen ambigüedad en los cálculos numéricos. Por ejemplo, SUM(columna) ignorará los valores nulos, lo que puede dar totals inexactos. COUNT(columna) no contará filas nulas, distorsionando recuentos.
  • Combinar valores nulos con otros datos válidos en joins, agrupaciones y cálculos puede propagar los valores nulos resultado en más datos nulos o información incorrecta.
  • Los valores nulos requieren manejo especial en la lógica del aplicativo para evitar errores. Verificar si un valor es nulo antes de utilizarlo en cálculos o tomar decisiones es crucial.
  • Índices y optimización de consultas funcionan mejor con menos valores nulos. Una alta cardinalidad de nulos hace más lentas las búsquedas indexadas.

Cómo los valores nulos pueden afectar la integridad de los datos

Rompen suposiciones sobre unicidad e identificadores: Claves primarias y otros identificadores como números de cédulas pueden dejar de ser únicos si se permiten nulos.

  • Invalidan restricciones CHECK y FOREIGN KEY: Las restricciones esperan un valor, si hay un nulo fallan las constrains.
  • Generan inconsistencia en aggregate functions: COUNT ignora filas nulas, SUM omite los nulos al sumar. Esto da diferentes resultados dependiendo del aggregate usado.
  • Causan ambigüedad en joins e inequality conditions: No está definido si NULL = NULL, o NULL > 10. Esto afecta joining tablas relacionadas.
  • Resultan en datos incompletos: Impide obtener la imagen completa de los datos si hay valores faltantes.

Funciones y Operadores para trabajar con valores Nulos

  • SQL provee varias funciones y operadores para manejar valores nulos:
  • COALESCE: Devuelve el primer valor no nulo de una lista de expresiones. Muy útil para sustituir nulos.
  • NVL: Similar a COALESCE, sustituye valores nulos en expresiones por un valor reemplazo especificado por el usuario.
  • NULLIF: Devuelve NULL si dos expresiones son iguales. Útil para anular valores válidos mapeándolos a nulo bajo ciertas condiciones.
  • CASE: Construye expresiones condicionales que pueden devolver valores nulos o no nulos según lógica compleja.
  • IS NULL / IS NOT NULL: Compara si una expresión es evaluada como un valor nulo. Se usa mucho en WHERE, JOIN, etc.
  • NVL2: Devuelve un valor si la expresión es no nula, o un valor alternativo si es nula.
